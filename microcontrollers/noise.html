Шуми в електронних
колах

Небажані випадкові сигнали, що накладаються на корисний. Шуми обмежують чутливість АЦП, спотворюють дані з датчиків і заважають надійній комунікації — їх потрібно розуміти і вміти боротися з ними.

learnt Electronics ADC STM32 ESP32
01 — Визначення

Шум (Noise) — це будь-яке небажане електричне коливання, що накладається на корисний сигнал і спотворює його. Шум є фундаментальним обмеженням будь-якої електронної системи: він не може бути усунутий повністю — лише мінімізований.

У embedded системах шум критично важливий для АЦП (аналого-цифровий перетворювач), сенсорних вимірювань, аудіо, радіо та будь-яких прецизійних вимірювань. Головний показник якості — відношення сигнал/шум (SNR, Signal-to-Noise Ratio): чим вище — тим краще.

KEY
Головна ідея: шум не можна усунути, але можна зменшити нижче рівня похибки системи. Стратегія: зрозуміти природу шуму → вибрати правильний метод боротьби → застосувати на потрібному рівні (апаратному або програмному).

04 — Спектр шумів — візуалізація
Спектральна щільність шуму vs Частота
1/f шум тепловий сума

На низьких частотах домінує 1/f шум — важливий для DC-вимірювань та сенсорів. На частотах вище кутової частоти (f_corner) починає переважати тепловий шум з рівним спектром. Сума обох дає реальний шум системи — синя крива.

Програмні (цифрові) методи
МетодПринципКоли використовувати
Усереднення (Averaging) Зчитати N вимірювань і взяти середнє. Випадковий шум зменшується в √N разів. Сенсори температури, тиску, будь-де де швидкість важлива менша за шум.
Ковзне середнє (Moving Avg) Середнє по останніх N вимірюваннях. Реалізується кільцевим буфером. Потоки даних в реальному часі де потрібне плавне значення. Затримка = N/2 вимірювань.
Фільтр Калмана Оптимальна оцінка стану з урахуванням моделі системи і шуму вимірювань. IMU, GPS-трекери, будь-де де є модель поведінки сигналу. Складніший, але найточніший.
Медіанний фільтр Впорядкувати N останніх значень і взяти середнє. Ефективно проти спайків. Захист від рідкісних різких викидів (ESD, button bounce). Нечутливий до outliers.
Oversampling + Decimation Зчитувати АЦП у 4× (або 16×) частіше → усереднювати. Кожні 4× вибірки дають +1 bit ENOB. Збільшити ефективну розрядність АЦП без зовнішніх компонентів. Вбудовано в STM32 HAL.
INF
Правило: спочатку апаратні методи, потім програмні. Не можна компенсувати поганий апаратний дизайн тільки алгоритмами — фільтри Калмана не замінять розв'язувальний конденсатор на VCC.
06 — Принцип Oversampling

Oversampling — техніка підвищення ефективної розрядності АЦП програмними засобами. Суть: зчитувати АЦП у 4N разів частіше ніж потрібно, потім підсумовувати і зсувати результат. Кожне подвоєння кількості вибірок (oversampling by 4) дає +1 біт ENOB.

Oversampling × 16 → +2 ENOB +2 bit
07 — Код

Три підходи до боротьби з шумом на практиці — від простого до ефективного.

C · STM32 HAL — Oversampling + Усереднення АЦП 
// ── Метод 1: просте усереднення (software averaging) ──
uint32_t adc_read_averaged(uint8_t samples) {
    uint32_t sum = 0;
    for (uint8_t i = 0; i < samples; i++) {
        HAL_ADC_Start(&hadc1);
        HAL_ADC_PollForConversion(&hadc1, 10);
        sum += HAL_ADC_GetValue(&hadc1);
    }
    return sum / samples;  // шум зменшується в √samples разів
}

// Виклик: 16 вибірок → шум / 4
uint32_t voltage = adc_read_averaged(16);


// ── Метод 2: ковзне середнє (moving average) ──
#define MA_SIZE 8
static uint16_t ma_buf[MA_SIZE] = {0};
static uint8_t  ma_idx = 0;
static uint32_t ma_sum = 0;

uint16_t moving_average(uint16_t new_val) {
    ma_sum -= ma_buf[ma_idx];   // прибрати старе значення
    ma_buf[ma_idx] = new_val;
    ma_sum += new_val;           // додати нове
    ma_idx = (ma_idx + 1) % MA_SIZE;
    return ma_sum / MA_SIZE;
}


// ── Метод 3: медіанний фільтр (захист від спайків) ──
uint16_t median_filter(uint16_t *buf, uint8_t n) {
    // простий bubble sort для невеликого n (3–9)
    uint16_t tmp[n];
    memcpy(tmp, buf, n * sizeof(uint16_t));
    for (uint8_t i = 0; i < n - 1; i++)
        for (uint8_t j = 0; j < n - i - 1; j++)
            if (tmp[j] > tmp[j+1]) {
                uint16_t t = tmp[j]; tmp[j] = tmp[j+1]; tmp[j+1] = t;
            }
    return tmp[n / 2];  // середнє значення після сортування
}
C · STM32 HAL — Апаратний Oversampling (вбудований в ADC) 
// STM32 HAL підтримує апаратний oversampling — не займає CPU
// Налаштовується через CubeMX: ADC → Oversampling → Enable

ADC_OversamplingTypeDef ovs;
ovs.Ratio          = ADC_OVERSAMPLING_RATIO_16;   // 16 вибірок
ovs.RightBitShift  = ADC_RIGHTBITSHIFT_4;         // ділимо на 16 = >> 4
ovs.TriggeredMode  = ADC_TRIGGEREDMODE_SINGLE_TRIGGER;
ovs.OversamplingStopReset = ADC_REGOVERSAMPLING_CONTINUED_MODE;

hadc1.Init.Oversampling = ovs;
hadc1.Init.OversamplingMode = ENABLE;
HAL_ADC_Init(&hadc1);

// Тепер HAL_ADC_GetValue() автоматично повертає усереднений результат
// 12-bit ADC + 16x oversampling → 14-bit ENOB
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, 10);
uint32_t result = HAL_ADC_GetValue(&hadc1);  // 14-bit результат
C · ESP-IDF v5.x — ADC з Multisampling 
// ESP32 ADC1 з усередненням через adc_oneshot
adc_oneshot_unit_handle_t adc1_handle;
adc_oneshot_unit_init_cfg_t init_config = {
    .unit_id = ADC_UNIT_1,
};
adc_oneshot_new_unit(&init_config, &adc1_handle);

adc_oneshot_chan_cfg_t chan_config = {
    .bitwidth   = ADC_BITWIDTH_12,
    .atten      = ADC_ATTEN_DB_11,   // 0–3.3V діапазон
};
adc_oneshot_config_channel(adc1_handle, ADC_CHANNEL_0, &chan_config);

// Програмне усереднення (ESP32 не має апаратного oversampling в oneshot)
int read_adc_averaged(int samples) {
    int sum = 0, val;
    for (int i = 0; i < samples; i++) {
        adc_oneshot_read(adc1_handle, ADC_CHANNEL_0, &val);
        sum += val;
    }
    return sum / samples;
}

09 — Питання на співбесіді

Тепловий шум (шум Джонсона-Найквіста) виникає через хаотичний тепловий рух носіїв заряду в будь-якому провіднику при температурі вище абсолютного нуля. Формула: V² = 4kTRΔf, де k — константа Больцмана, T — температура [K], R — опір [Ом], Δf — смуга [Гц].

Позбутись неможливо тому що це фундаментальне термодинамічне явище — поки T > 0K, носії заряду рухаються хаотично і створюють шумовий струм. Можна лише зменшити: знизити температуру (кріогенні детектори), зменшити опір, або звузити смугу пропускання.

SNR (Signal-to-Noise Ratio) — відношення корисного сигналу до шуму, виражене в децибелах: SNR = 20·log₁₀(Vsignal / Vnoise).

Для ідеального АЦП без зовнішніх шумів SNR визначається лише квантизаційним шумом: SNR = 6.02·N + 1.76 дБ, де N — розрядність. Наприклад: 12-bit → SNR ≈ 74 дБ. Кожен додатковий біт дає +6 дБ SNR.

На практиці реальний SNR завжди менший через тепловий і 1/f шум. Показник ENOB (Effective Number of Bits) показує реальну корисну розрядність: ENOB = (SINAD − 1.76) / 6.02. Типово 12-bit АЦП дає ENOB = 10–11 біт.

Oversampling — зчитування АЦП у 4N разів частіше, потім підсумовування і зсув вправо. Кожне зчитання містить шум, але оскільки шум випадковий — при підсумовуванні він частково компенсується (деструктивна інтерференція), а корисний сигнал складається.

Формула: ENOB_new = ENOB_original + log₂(N_oversample) / 2. Для ×16 oversampling: +log₂(16)/2 = +2 біти. 12-bit АЦП → 14-bit ENOB.

Умова: сигнал повинен містити шум амплітудою ≥ ½ LSB — без шуму вибірки однакові і oversampling нічого не дає. Якщо шуму немає — його штучно додають (dithering).

Кожна траса на платі має паразитну індуктивність (~1 нГн/мм). При різких змінах струму (перемикання цифрових виходів, початок передачі UART) виникає сплеск напруги: V = L · dI/dt.

Конденсатор — це локальний резервуар заряду. Якщо він стоїть далеко від VCC піна — між ним і MCU є індуктивна траса, яка уповільнює реакцію і послаблює ефект. Якщо стоїть впритул — заряд надходить миттєво і сплеск живлення гаситься до того як він "дістанеться" до АЦП або підсилювача.

Правило: 100 нФ (кераміка) не далі 2–3 мм від кожного VCC піна. 10 мкФ (електроліт або тантал) — один на групу живлення.

1/f шум (мерехтливий, рожевий шум) — шум, спектральна щільність якого обернено пропорційна частоті: S(f) ∝ 1/f. Виникає через дефекти кристалічної решітки напівпровідника — носії заряду "застрягають" і "вивільняються" на цих дефектах хаотично.

Критичний для: DC-вимірювань і дуже низькочастотних сигналів (сенсори температури, тиску, ваги), операційних підсилювачів з великим коефіцієнтом підсилення, мікрофонних підсилювачів.

Методи боротьби: вибирати компоненти з низькою кутовою частотою (f_corner) — там де 1/f = тепловий шум. JFET-вхідні ОП зазвичай кращі за CMOS для низькошумових застосувань.

Цифрова частина MCU споживає струм імпульсами при кожному перемиканні логіки. Ці імпульсні струми протікають через паразитний опір і індуктивність доріжок землі — і створюють мікровольтові сплески на GND.

Якщо аналоговий тракт (підсилювач, АЦП) ділить ту ж землю — ці сплески потрапляють як шум у корисний сигнал. 1 мВ шуму на землі при підсиленні 100× → 100 мВ шуму на виході — це вже помітне спотворення.

Рішення: окрема площина AGND для аналогової частини, з'єднана з DGND в одній точці (зірка). Цифрові сигнали не проходять під аналоговими трасами. На STM32 пін VREF+ живиться від окремого стабілізатора або через RC-фільтр.

Ковзне середнє (Moving Average) — бере середнє арифметичне N останніх вимірювань. Ефективно зменшує рівномірний (гаусівський) шум, але чутливий до одиночних викидів (spike): один спайк "живе" у вікні N тактів і впливає на всі N результатів.

Медіанний фільтр — сортує N значень і бере середнє за індексом. Повністю ігнорує одиночні викиди — вони опиняються на краях відсортованого масиву і не потрапляють у медіану. Але гірше зменшує рівномірний шум ніж averaging.

Правило вибору: якщо є ризик рідкісних різких спайків (ESD, механічний контакт, button bounce) → медіанний. Якщо шум рівномірний і безперервний (тепловий, АЦП) → ковзне середнє або oversampling.
